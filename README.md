# MathViz Architect 📐

**Приложение для развития визуального мышления на уроках математики**

Интерактивное desktop-приложение для создания математических визуализаций, диаграмм и обучающих материалов для учеников 5-11 классов.

## 🎯 Возможности

### 📊 Инструменты рисования
- **Базовые фигуры**: прямоугольники, круги, треугольники
- **Математические фигуры**: окружность, треугольник, четырёхугольник с редактируемыми сторонами
  - Все фигуры прозрачные (только контур), стороны задаются в панели свойств
  - Треугольник: валидация неравенства треугольника с предупреждением
  - Четырёхугольник: стороны AB/BC/CD/DA с автоматическим пересчётом размера
- **Линия**: рисование прямых линий drag-to-draw с опциональными стрелками на концах
- **Ластик**: удаление объектов кликом или проведением мыши
- **Текст**: редактирование по двойному клику
- **Диаграммы**: столбчатые и круговые диаграммы с редактированием данных

### 🎯 Выделение и редактирование
- **Marquee selection**: выделение нескольких объектов областью (drag на пустом месте)
- **Multi-select**: Ctrl+клик для добавления/снятия объектов из выделения
- **Multi-drag**: перетаскивание всех выбранных объектов одновременно
- **Backspace в тексте**: корректное удаление символов при редактировании текста

### 🎓 Интерактивные модули (18 модулей)

Все модули имеют профессиональный двухколоночный layout с графиком слева и панелью управления справа, встроенные инструкции и tooltips для элементов управления.

#### Функции (7 модулей)
- **Линейная функция** (y = kx + b) - интерактивное изменение параметров
- **Квадратичная функция** (y = ax² + bx + c) - визуализация параболы с вершиной, плавная цикличная анимация
- **Показательная функция** (y = a·bˣ + c) - экспоненциальный рост и убывание
- **Логарифмическая функция** (y = a·log_b(x) + c) - обратная к показательной
- **Производная** - исследование функции y = ax³ + bx² + cx + d, экстремумы, точки перегиба
- **Пересечения функций** - нахождение точек пересечения двух графиков
- **Преобразования функций** - визуализация сдвигов, растяжений и отражений графиков (y = k·f(d·x - a) + c)
- **Исследование функции** - полный анализ кубической функции: критические точки, экстремумы, точки перегиба, монотонность, выпуклость

#### Геометрия (5 модулей)
- **Теорема Пифагора** - анимированное доказательство с квадратами, пошаговые объяснения, динамическая формула
- **Векторы** - операции с векторами (сложение, вычитание, скалярное произведение, угол)
- **Координатная плоскость** - добавление точек кликом, изучение координат
- **Параллельные прямые** - углы при пересечении параллельных прямых секущей, корректная визуализация углов
- **Подобие треугольников** - свойства подобных треугольников и коэффициент подобия

#### Тригонометрия (3 модуля)
- **Тригонометрическая окружность** - sin, cos, tg с анимацией, переключение градусы/радианы
- **Тригонометрические функции** - графики sin, cos, tan, cot с изменением амплитуды, периода и фазы, корректная обработка асимптот
- **Окружность и график** - анимированная визуализация связи единичной окружности и синусоиды/косинусоиды в реальном времени

#### Алгебра (3 модуля)
- **Вероятностное дерево** - визуализация условных вероятностей для 2-3 событий
  - Два режима: простое дерево и условные вероятности
  - Переименование событий
  - Динамическое добавление третьего уровня
  - Автоматический расчёт итоговых вероятностей
- **Системы неравенств** - визуализация решений систем линейных неравенств

## 🎮 Режим задач (v2.0.0) — архитектура уровня Khan Academy

Полноценная адаптивная система обучения на основе шаблонов — один шаблон генерирует сотни уникальных задач.

### Архитектура (8 слоёв)

```
Curriculum → TopicGraph → ProblemSelector → ProblemTemplate → VariantGenerator → SolutionEngine → AssessmentEngine → AdaptiveEngine
```

**Ключевые файлы:**

| Файл | Описание |
|------|----------|
| `src/lib/types.ts` | Типы `ProblemTemplate`, `DifficultyConfig`, `GeneratedProblem`, `SolutionStep`, `ParameterDef` |
| `src/lib/curriculum.ts` | Структура учебной программы: классы, предметы, темы |
| `src/lib/topicGraph.ts` | Граф зависимостей между темами (prerequisites) |
| `src/lib/problemSelector.ts` | Выбор шаблона с учётом истории сессии (`ProblemSession`) |
| `src/lib/templateEngine.ts` | Facade-реэкспорт (backward compatibility) |
| `src/lib/engine/expressionParser.ts` | CSP-safe парсер выражений |
| `src/lib/engine/variantGenerator.ts` | Генерация параметров и вариантов задач |
| `src/lib/engine/answerValidator.ts` | Проверка ответа пользователя |
| `src/lib/engine/mistakeAnalyzer.ts` | Анализ типичных ошибок |
| `src/lib/problemTemplates.ts` | 19 multi-difficulty шаблонов для 5 класса (49 конфигураций) |
| `src/lib/adaptiveEngine.ts` | Адаптивный движок сложности |
| `src/components/challenge/ChallengeMode.tsx` | UI режима задач |

### 🔄 Цикл обучения (Learning Loop)

Каждая задача проходит следующий цикл:

```
Topic
 ↓
ProblemSelector (учитывает сессию, избегает повторов)
 ↓
Генерация варианта (VariantGenerator)
 ↓
Ответ пользователя
 ↓
Проверка (AssessmentEngine)
 ↓
Обратная связь
 ↓
Обновление сложности (AdaptiveEngine)
 ↓
Обновление mastery + ProblemSession
 ↓
Следующая задача
```

### 🔁 Pipeline задачи (Problem Flow)


### 🧩 Структура шаблона задачи

```typescript
type AnswerType =
  | 'number'      // ✅ реализовано — большинство задач 5–7 классов
  | 'fraction'    // ✅ реализовано — дроби (6–7 класс)
  | 'coordinate'  // ✅ реализовано — координатная плоскость (6–7 класс)
  | 'expression'  // 🔜 зарезервировано — 7+ класс
  | 'interval'    // 🔜 зарезервировано — 8+ класс
  | 'set'         // 🔜 зарезервировано — 9+ класс

interface DifficultyConfig {
  template: string            // текст задачи с {параметрами}
  parameters: ParameterDef[]  // определения параметров
  answer_formula: string      // формула для вычисления ответа
  answer_type?: AnswerType    // default: 'number'
  constraints?: string[]      // ограничения на параметры
  solution?: SolutionStep[]   // пошаговое решение
  hint?: string               // подсказка
  common_mistakes?: CommonMistake[]  // типичные ошибки → умная обратная связь
}

interface ProblemTemplate {
  id: string
  class: number
  subject: string
  section: string
  topic: string
  topic_title: string
  problemType: 'numeric' | 'comparison' | 'text'

  difficulties: Partial<Record<1 | 2 | 3 | 4, DifficultyConfig>>

  relatedModule?: string      // ссылка на интерактивный модуль
}
```

`variantGenerator` автоматически выбирает ближайший доступный уровень сложности если запрошенный отсутствует (fallback вниз, потом вверх).

### 📦 Структура сгенерированной задачи

```typescript
interface GeneratedProblem {
  id: string                          // уникальный ID экземпляра
  template_id: string                 // ссылка на шаблон
  params: Record<string, number | string>  // подставленные параметры
  question: string                    // текст задачи с подставленными значениями
  answer: number | string             // правильный ответ
  answer_type: AnswerType             // тип ответа
  hint?: string                       // подсказка с подставленными значениями
  solution?: SolutionStep[]           // пошаговое решение
}
```

### 🎯 ProblemSession (сессия пользователя)

```typescript
interface ProblemSession {
  topic: string             // текущая тема
  currentDifficulty: number // текущий уровень сложности
  streak: number            // серия правильных ответов
  recentTemplateIds: string[] // последние 5 шаблонов (для избежания повторов)
  accuracy: number          // точность за последние 10 задач
}
```

`ProblemSelector` использует сессию чтобы не повторять недавно показанные шаблоны и выбирать шаблон подходящей сложности.

### 💥 Variant Explosion

Один шаблон порождает сотни уникальных вариантов на каждом уровне сложности:

```
Пример: grade5-perimeter
  difficulty 1: a ∈ [3..15], b ∈ [3..15]  → 169 вариантов
  difficulty 2: a ∈ [3..20], b ∈ [3..20]  → 324 варианта

19 шаблонов × 3 уровня × ~200 вариантов → ~11 400 задач (только 5 класс)
```

### 🗺️ Topic Progression (прохождение тем)

Темы связаны в граф зависимостей — `topicGraph.ts`:

```
Сложение
   ↓
Умножение
   ↓
Порядок действий
   ↓
Выражения
```

- Мягкие зависимости — тема остаётся доступной, но показывается предупреждение «Рекомендуем сначала пройти: X»
- Предупреждение автоматически исчезает после достижения уровня `proficient`/`mastered`

### 🧠 Адаптивный алгоритм (`adaptiveEngine.ts`)

| Условие | Изменение сложности |
|---------|---------------------|
| 3 правильных подряд | +1 |
| 3 ошибок подряд | -1 |
| Accuracy < 40% (10 задач) | -1 |
| Accuracy > 80% (10 задач) | +1 |

Ограничение: `difficulty ∈ [1, 4]`

Текущая сложность отображается в UI рядом со streak.

### 🔭 Задача → Интерактивный модуль

Уникальная фича: шаблоны задач напрямую связаны с интерактивными модулями через поле `relatedModule`.

```
Задача: «Угол между параллельными прямыми»
   → кнопка «Исследовать»
   → открывает модуль ParallelLines
```

Сейчас реализовано для 11 шаблонов (геометрия, уравнения). Поле `relatedModule` поддерживается всеми шаблонами — расширяется по мере добавления новых интерактивных модулей.

> 💡 **Это уникальная фича проекта.** Большинство тренажёров — просто тесты. MathViz Architect превращает каждую задачу в точку входа в интерактивную математическую лабораторию. Ученик не просто решает — он исследует.

## 📝 Content Authoring Guide

Стандарт для авторов задач. Соблюдение этих правил обязательно — иначе при 100+ шаблонах начнётся хаос.

### Обязательные поля шаблона

Каждый шаблон **обязан** содержать:

```typescript
{
  id: 'grade{N}-{topic}',             // уникальный ID, например: grade6-fractions
  class: 6,
  subject: 'algebra' | 'geometry' | 'logic',
  section: 'Название раздела',
  topic: 'camelCase',
  topic_title: 'Читаемое название',
  problemType: 'numeric' | 'comparison' | 'text',

  difficulties: {
    1: {
      template: 'Текст задачи с {параметрами}',
      parameters: { ... },
      answer_formula: 'выражение',
      hint: 'Подсказка',
    },
    2: {
      template: 'Более сложный вариант',
      parameters: { ... },   // больший диапазон
      answer_formula: 'выражение',
    },
  },
}
```

### Рекомендуемые поля внутри DifficultyConfig

```typescript
{
  solution: [
    { explanation: 'Шаг 1...' },
    { expression: '{a} + {b}', result: '{answer}' },
  ],
  common_mistakes: [
    { pattern: 'a + b', feedback: 'Объяснение ошибки' },
  ],
}
```

`relatedModule` задаётся на уровне шаблона (не внутри difficulty):

```typescript
{
  id: 'grade5-perimeter',
  ...
  difficulties: { ... },
  relatedModule: 'coordinate-plane',
}
```

### Соглашения по ID

| Паттерн | Пример |
|---------|--------|
| `grade{N}-{topic}` | `grade6-fractions` |
| topic — латиница camelCase | `fractions`, `linearEq`, `pythagorean` |
| Один шаблон на тему, уровни внутри `difficulties` | — |

### Правила параметров

- Минимальные значения: не используйте `0` там, где он вызывает деление или бессмысленный результат
- Добавляйте `constraints` для исключения вырожденных случаев: `['a !== b', 'a + b < 100']`
- Параметры-выражения (`type: 'expression'`) вычисляются после всех `int`/`choice` параметров

### Уровни сложности — что меняется

| difficulty | Что делать |
|------------|------------|
| 1 | Малые числа (до 20), одно действие |
| 2 | Числа до 100, 2 действия |
| 3 | Составные задачи, несколько шагов |
| 4 | Олимпиадный уровень, нестандартный подход |

### Типы ответов (`answer_type`)

| Тип | Статус | Пример ответа | Принимаемые форматы |
|-----|--------|---------------|---------------------|
| `number` | ✅ дефолт | `24` | `24`, `24.0`, `"24 "` |
| `fraction` | ✅ | `3/4` | `3/4`, `3 / 4`, `0.75` |
| `coordinate` | ✅ | `(3, 4)` | `(3, 4)`, `3, 4`, `3;4` |
| `expression` | 🔜 8+ класс | `2x + 1` | — |
| `interval` | 🔜 8+ класс | `(-∞, 5]` | — |
| `set` | 🔜 9+ класс | `{1, 2, 3}` | — |

UI автоматически показывает подсказку в поле ввода в зависимости от типа.

### ⚙️ Архитектура `engine/` (рефактор выполнен)

`templateEngine.ts` разделён на 4 специализированных модуля:

```
src/lib/engine/
  expressionParser.ts   ← CSP-safe парсер выражений
  variantGenerator.ts   ← генерация случайных параметров
```

`src/lib/templateEngine.ts` остаётся как facade для обратной совместимости — все существующие импорты работают без изменений.

## 🎨 Режимы работы

### 1. Режим выбора (Select)
- Выбор и перемещение объектов на холсте
- **Marquee selection**: выделение нескольких объектов областью (зажать и тянуть по пустому месту)
- **Multi-select**: Ctrl+клик (или Cmd+клик на Mac) для добавления/снятия объектов
- **Multi-drag**: перетаскивание всех выбранных объектов одновременно

### 2. Режим рисования (Draw)
- **Фигуры**: прямоугольники, круги, треугольники
- **Линия**: рисование прямых линий drag-to-draw с опциональными стрелками (настраивается в панели свойств)
- **Ластик**: удаление объектов кликом или проведением мыши
- **Текст**: добавление текстовых блоков (Backspace корректно работает при редактировании)
- **Диаграммы**: столбчатые и круговые диаграммы с редактором данных

### 3. Интерактивные модули (Interactive)
18 готовых математических визуализаций с параметрами:
- Функции (линейная, квадратичная, показательная, логарифмическая, производная, пересечения, преобразования, исследование)
- Геометрия (теорема Пифагора, векторы, координатная плоскость, параллельные прямые, подобие треугольников)
- Тригонометрия (тригонометрическая окружность, тригонометрические функции, окружность и график)
- Алгебра (вероятностное дерево, системы неравенств)

### 4. Библиотека шаблонов (Library)
Готовые шаблоны для быстрого создания инфографики

### 5. Режим задач (Challenge)
Математические задачи с проверкой ответов для 5-11 классов

## 🌐 Деплой на GitHub Pages

Проект настроен для автоматического деплоя на GitHub Pages через GitHub Actions.

### Автоматический деплой

При каждом push в ветку `main` автоматически запускается workflow, который:
1. Устанавливает зависимости
2. Собирает production-версию
3. Деплоит на GitHub Pages

### Настройка (один раз)

1. **Включите GitHub Pages в настройках репозитория:**
   - Перейдите в Settings → Pages
   - Source: выберите "GitHub Actions"

2. **Проверьте права доступа:**
   - Settings → Actions → General → Workflow permissions
   - Выберите "Read and write permissions"

3. **После первого push в main:**
   - Workflow запустится автоматически
   - Приложение будет доступно по адресу: `https://[username].github.io/MathViz-Architect/`

### Ручной деплой

1. Перейдите в Actions → Deploy to GitHub Pages
2. Нажмите "Run workflow"
3. Выберите ветку `main` и нажмите "Run workflow"

### Локальная сборка для GitHub Pages

Чтобы протестировать сборку локально:

```bash
cd mathviz-architect
GITHUB_PAGES=true pnpm build
pnpm preview
```

## 🤝 Совместная работа

### Для нового разработчика:

1. **Клонируйте репозиторий**
```bash
git clone <repository-url>
cd mathviz-architect
```

2. **Установите зависимости**
```bash
pnpm install
```

3. **Запустите dev-сервер**
```bash
pnpm run dev
```

4. **Создайте новую ветку для фичи**
```bash
git checkout -b feature/my-new-feature
```

5. **Внесите изменения и закоммитьте**
```bash
git add .
git commit -m "Add: описание изменений"
git push origin feature/my-new-feature
```

### Соглашения о коммитах:
- `Add:` - новая функциональность
- `Fix:` - исправление бага
- `Update:` - обновление существующей функциональности
- `Refactor:` - рефакторинг кода
- `Docs:` - изменения в документации

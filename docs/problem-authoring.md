# Problem Authoring Guide

Стандарт для авторов задач MathViz Architect.  
Соблюдение обязательно — иначе при 100+ шаблонах начнутся расхождения в стиле.

---

## 1. Naming Convention

### ID шаблона

```
grade{N}-{topic}
```

| Часть | Правило | Пример |
|-------|---------|--------|
| `grade{N}` | номер класса | `grade6` |
| `{topic}` | латиница, camelCase | `fractions`, `linearEq`, `pythagorean` |

**Примеры:**
```
grade5-perimeter     ✅
grade6-fractions     ✅
grade7-linearEq      ✅

grade5-perimeter-1   ❌  (порядковый номер не нужен — уровни внутри difficulties)
grade5_perimeter     ❌  (нижнее подчёркивание)
grade5-Perimeter     ❌  (заглавная буква)
```

### topic (ключ темы)

Должен совпадать с ключом в `curriculum.ts`. Проверяй перед добавлением.

### topic_title (отображаемое название)

- На русском языке
- Без точки в конце
- Конкретно и ясно

```
'Периметр прямоугольника'    ✅
'Нахождение периметра'       ❌  (слишком общее)
'Периметр.'                  ❌  (точка)
```

## 2. Difficulty Guidelines

Каждый шаблон должен покрывать **минимум 2 уровня сложности**.

### difficulty 1 — Легко

- Одно действие
- Числа ≤ 20
- Очевидная структура задачи
- Обязательно: `hint`

```typescript
// Пример параметров для diff 1
parameters: {
  a: { type: 'int', min: 2, max: 10 },
  b: { type: 'int', min: 2, max: 10 },
}
```

### difficulty 2 — Средне

- 2 шага решения
- Числа ≤ 100
- Может потребоваться промежуточное действие

```typescript
// Пример параметров для diff 2
parameters: {
  a: { type: 'int', min: 5, max: 20 },
  b: { type: 'int', min: 5, max: 20 },
}
```

### difficulty 3 — Сложно

- 3+ шагов решения
- Числа ≤ 1000 или дробные значения
- Обратная задача (найти сторону по площади и т.п.)
- Обязательно: `solution[]`

### difficulty 4 — Олимпиадное

- Нестандартный подход
- Требует инсайта, не только алгоритма
- Обязательно: `solution[]` с полным разбором

## 3. Parameter Rules

### Базовые правила

- Никогда не используй `min: 0` если параметр — длина, знаменатель или делитель
- Минимальное значение для геометрии: `min: 2`
- Минимальное значение для знаменателей дробей: `min: 2`

### Красивые ответы

Генератор должен давать ответы которые удобно вводить.  
Проверяй через `pnpm test:templates`.

**Не допускается:**
```
0.333333...    ❌  (бесконечная дробь)
17/19          ❌  (некрасивая дробь, знаменатель > 12)
-0.00001       ❌  (отрицательный или около нуля)
```

**Допустимо:**
```
0.5, 0.25, 0.75    ✅  (простые десятичные)
1/2, 3/4, 2/3      ✅  (знаменатель ≤ 12)
целые числа        ✅  (всегда)
```

### Constraints

Используй `constraints` для исключения вырожденных случаев:

```typescript
constraints: ['a !== b']           // стороны не равны
constraints: ['a > b']             // a строго больше
constraints: ['a + b < 100']       // сумма в разумных пределах
constraints: ['a % b === 0']       // делится нацело
```

### Параметры-выражения

Параметры типа `expression` вычисляются **после** всех `int` и `choice`.  
Используй их для зависимых величин:

```typescript
parameters: {
  a: { type: 'int', min: 2, max: 10 },
  b: { type: 'int', min: 2, max: 10 },
  s: { type: 'expression', value: 'a * b' },  // площадь через стороны
}
```

## 4. Solution Step Style

Поле `solution` обязательно для difficulty 3 и 4, рекомендуется для 1 и 2.

### Структура шага

```typescript
solution: [
  { explanation: 'Текст объяснения шага' },
  { expression: 'математическое выражение', result: '{answer}' },
]
```

### Правила написания

**explanation** — полное предложение, без сокращений:
```
'Периметр прямоугольника — сумма всех четырёх сторон'   ✅
'P = сумма сторон'                                       ❌
```

**expression** — математическая запись, параметры в фигурных скобках:
```typescript
expression: '2 × ({a} + {b})'    ✅
expression: '2*(a+b)'             ❌  (без подстановки)
```

**result** — итог шага, обычно `'{answer}'` или промежуточное значение:
```typescript
result: '{answer} см'     ✅  (с единицей измерения)
result: '{answer}'        ✅  (без единицы — тоже ок)
```

### Типовая структура для геометрии (3 шага)

```typescript
solution: [
  { explanation: 'Запишем формулу периметра прямоугольника:' },
  { explanation: 'Подставим значения:', expression: '2 × ({a} + {b})' },
  { explanation: 'Вычислим:', expression: '2 × {a} + 2 × {b}', result: '{answer} см' },
]
```

## 5. Common Mistakes Style

Поле `common_mistakes` показывает умную обратную связь вместо простого "Неправильно".

### Структура

```typescript
common_mistakes: [
  { pattern: 'выражение', feedback: 'Объяснение ошибки' },
]
```

### Правила

**pattern** — формула которую вычисляет движок и сравнивает с ответом пользователя:
```typescript
{ pattern: 'a + b', feedback: '...' }       // если ввёл a+b вместо 2*(a+b)
{ pattern: 'a * b', feedback: '...' }       // если ввёл площадь вместо периметра
{ pattern: '2 * a', feedback: '...' }       // если умножил только одну сторону
```

**feedback** — объяснение что пошло не так + правильное направление:
```
'Ты сложил стороны только один раз. Периметр = 2 × (a + b).'   ✅
'Неправильно. Попробуй ещё раз.'                                ❌  (не информативно)
'Это формула площади, а не периметра.'                          ✅
```

### Типичные ошибки по темам

| Тема | Ошибка | pattern |
|------|--------|---------|
| Периметр | забыл умножить на 2 | `a + b` |
| Периметр | перепутал с площадью | `a * b` |
| Площадь | перепутал с периметром | `2 * (a + b)` |
| Угол треугольника | сумма двух углов | `a + b` |
| Уравнение x+a=b | не вычел, а сложил | `a + b` |

## 6. answer_type

Указывай явно если ответ не число:

```typescript
answer_type: 'fraction'      // для задач с дробями
answer_type: 'coordinate'    // для задач с координатами
// answer_type: 'number'     // дефолт, можно не писать
```

Зарезервированы для 8–11 классов: `expression`, `interval`, `set`.

```
[ ] topic совпадает с ключом в curriculum.ts
[ ] Минимум 2 уровня сложности в difficulties
[ ] Нет min: 0 у геометрических параметров
[ ] Добавлены constraints для исключения вырожденных случаев
[ ] hint заполнен для difficulty 1
[ ] solution заполнен для difficulty 3 и 4
[ ] Запущен pnpm test:templates — все проверки прошли
```
